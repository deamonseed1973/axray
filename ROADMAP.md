# AXray Roadmap

## v1 (shipped ✅)
- AX tree walking with visitor pattern (inspired by AXorcist)
- Parallel Vision pipeline — OCR + barcode on one `VNImageRequestHandler` (inspired by Viz)
- Three CLI commands: `inspect`, `find`, `audit`
- JSON output mode for scripting
- GitHub Actions macOS CI

**Verdict: ⭐⭐⭐ 3/5** — solid foundation, needs richer output and documentation

---

## v2 Roadmap

### 1. Image Export (`--save-images`)
Every element that gets a Vision analysis pass should also export a screenshot of its captured frame.

- `--save-images <dir>` flag on `inspect` and `audit` commands
- Save as `<role>-<identifier>-<index>.png` per element
- Include a manifest JSON linking each image to its analysis result
- Particularly useful for audit reports — attach the screenshot to each accessibility gap/mismatch found

### 2. Richer Documentation
v1 README was thin. v2 needs:

- **Worked examples** with actual terminal output (use asciinema or embedded code blocks)
- **Expected output format** documented for each command, with annotated examples
- **Permissions setup guide** — how to grant Accessibility + Screen Recording to the binary
- **Use case walkthroughs**: testing an app for accessibility gaps, finding unlabelled buttons, auditing a custom view
- Man page / `--help` output improvements with longer descriptions and examples

### 3. LLM Bug Report Pipeline (`axray report`)
New command: `axray report <app> [--model <openai|ollama|anthropic>] [--output report.md]`

The idea: pipe the structured audit output (accessibility gaps + label mismatches + element screenshots) into an LLM to produce a human-readable, actionable accessibility bug report.

**Pipeline:**
```
axray audit → structured JSON → LLM prompt → bug report (Markdown/HTML)
```

**Report format per issue:**
- Element path and role
- What Vision detected vs what AX reports
- Screenshot of the element (base64 embedded or file reference)
- Suggested fix (generated by LLM)
- Severity rating (LLM-assessed: critical/major/minor)

**LLM backends to support:**
- OpenAI (`gpt-4o`, `gpt-4-vision`) — can receive screenshots directly
- Anthropic (Claude) — native vision support
- Local/Ollama — text-only fallback (no image embedding)

**LLM prompt strategy:**
Feed the structured JSON plus element screenshots, ask for:
1. A plain-English description of each accessibility problem
2. Why it matters (which users it affects: VoiceOver, Switch Control, etc.)
3. Specific code-level fix suggestion for the developer
4. A severity rating

### 4. Structured Output Improvements
- `--format sarif` — output as SARIF (Static Analysis Results Interchange Format) for CI integration
- `--format junit` — output as JUnit XML for test runner integration
- Exit code 1 when audit finds issues (enables CI fail-on-accessibility-gaps)

### 5. Watch Mode (`axray watch`)
- Monitor a running app continuously; re-scan when the focused window changes
- Stream new issues as they're detected (useful during development)

### 6. Filtering & Focus
- `--role <AXRole>` — limit inspection to specific element types
- `--min-size <WxH>` — skip tiny elements unlikely to have meaningful visual content
- `--only-gaps` — audit only shows elements with no AX text that Vision can read
